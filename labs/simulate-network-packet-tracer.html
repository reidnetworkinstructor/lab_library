Simulate a Network with Packet Tracer

Design a basic network with router, switch, and firewall for hands-on learning

Objective

Use Cisco Packet Tracer to build a network with a router, switch, and firewall, exploring their roles for Network+ readiness.

Tool Needed: Cisco Packet Tracer (free from netacad.com after signing up).

Step 1: Set Up Your Network

  • Open Packet Tracer and drag in:
    • 1 Router (e.g., 1941)
    • 1 Switch (e.g., 2950-24)
    • 2 PCs (End Devices)
    • 1 Firewall (use ASA 5506-X or similar)
  • Connect the devices:
    • Router → Firewall (GigabitEthernet0/0 to 0/1)
    • Firewall → Switch (0/2 to port 1)
    • PCs → Switch (ports 2 and 3)
  • Use copper straight-through cables for all connections.

Step 2: Configure the Router

Click the router, go to the CLI tab, and input appropriate IP and routing settings (you may simulate an external connection here).


enable
configure terminal
interface GigabitEthernet0/0
ip address 203.0.113.1 255.255.255.0
no shutdown
exit

This sets the router to connect to the “internet” (simulated).

Step 3: Configure the Firewall

Click the firewall device, go to CLI, and enter a basic access rule:


enable
configure terminal
access-list allow_all extended permit ip any any
interface Ethernet0/1
nameif outside
security-level 0
ip address 203.0.113.2 255.255.255.0
no shutdown
exit
interface Ethernet0/2
nameif inside
security-level 100
ip address 192.168.2.1 255.255.255.0
no shutdown
exit
access-group allow_all in interface inside

This links the router to the switch and allows basic traffic.

Step 4: Configure PCs and Switch

  • PC0: IP: 192.168.2.10, Mask: 255.255.255.0, Gateway: 192.168.2.1
  • PC1: IP: 192.168.2.11, Mask: 255.255.255.0, Gateway: 192.168.2.1
  • Switch: No configuration needed (plug-and-play).

Step 5: Test and Reflect

  • On PC0, open Command Prompt and ping 192.168.2.11 (PC1)—confirm switch is working.
  • Then ping 203.0.113.1 (router external)—confirm firewall and router are passing traffic.

Reflection: Write a short paragraph. How did each device contribute to the flow of traffic?

Hint: Switch connects internal devices, firewall applies rules and segmentation, router handles outbound routing to the “internet.”
